## 💰 費用の総覧

Lasso 本体は **完全無料・MIT オープンソース** です。各機能の実際のコストは以下の通り：

| 機能 | 費用 | 備考 |
|---|---|---|
| Lasso 本体（MCP サーバー + すべてのコア機能） | ✅ 無料 | MIT オープンソース、永久無料 |
| 検索（Zhipu + Brave + Bing） | ✅ 無料枠あり | Zhipu はトークン課金、Brave は **月 2,000 件無料**、Bing は **月 1,000 件無料**——払わなくても使えます |
| 公開ページのスクレイプ / スクリーンショット / PDF / ネットワーク監査 / 生バイト取得 | ✅ 無料 | ローカルで動作、キー不要、支払い不要 |
| ログイン済みページのスクレイプ（ローカル Chrome 再利用） | ✅ 無料 | ローカルで動作、キー不要、支払い不要 |
| デスクトップ操作（macOS / Windows / Linux） | ✅ 無料 | ローカルでビルド・実行、OS の認可だけが必要。署名付きの永続認可には Apple Developer アカウント \$99/年が **任意**（未署名でも動作——その都度再認可するだけ） |
| クラウドブラウザ（browserbase / stagehand） | ⚠️ 有料・デフォルトはオフ | トライアル以降は有料。**設定しなければ一切課金されません**——Lasso 唯一の有料項目です |

> 一言で言えば：**クラウドブラウザを有効にしない限り、Lasso のコストはゼロ** です——検索には日常使いに十分な無料枠があり、それ以外はすべて完全無料です。

## 60秒ではじめる

### 30秒 · 一行インストール（設定ゼロ）

```bash
claude mcp add lasso -- npx -y lasso-mcp
```

Claude Code を再起動 → `/mcp` → `lasso ✓ Connected` と表示されれば完了。**インストールコマンドにキーは一切不要**——設定は別ステップ（次の層）で行います。

### 30秒 · 設定なしでもう既にできること

インストール直後、キーがなくても以下はすべて使えます（**ティア 1：設定ゼロ**）：

- **公開ウェブページ**のテキストを取得し、整った markdown に変換
- **ページ全体のスクリーンショット** と **PDF 保存**（ファイルパスを返す）
- ページが読み込む **サードパーティのトラッカー** を確認
- JSON API やファイルから生バイトを直接取得
- macOS のネイティブアプリを操作（Finder / Mail / システム設定など——システム設定で一度だけチェックが必要）

最初の一歩——Claude にこう言うだけ：

> 「example.com のテキストを取得して markdown にして」

### もっと欲しい？ 設定ファイルで追加（ティア 2）

- **検索** → `lasso config init` を実行して `~/.lasso/config.json` を作成し、Zhipu のキーを記入（[設定](#設定) を参照）
- **ログイン済みページのスクレイプ**（Jira / プライベート GitHub / 社内ネット） → `lasso launch-chrome` を一度だけ実行
- **macOS デスクトップの操作** → `lasso doctor` を一度だけ実行して認可を案内してもらう

各キーの取得方法や無料枠の詳細は [**キー設定ガイド**](./doc/KEY-GUIDE.md) を参照してください。

## 何ができるか

**ツール名**ではなく、**やりたいこと**ごとに整理しています。どれも一言で入力し、一言で結果が返ります。

### 検索

> あなた：「X を検索して」→ 構造化された検索結果

デフォルトは Zhipu（中国語に強い）。Brave と Bing を追加してマルチソース化も可能。**いずれかのソースがレート制限や障害で使えなくなっても、自動で次に切り替わります——ユーザーは何も感じません。** 一つのプロバイダーの無料枠を使い切っても、全体が止まることはありません。

### 公開ページのスクレイプ（ログイン不要）

> あなた：「example.com のテキストを取得して」→ 整った記事テキスト（3 段階の粒度を選択可能）

ナビバーや広告、サイドバーなどの雑多な要素を自動で除去——**トークンを 30〜70% 節約**（費用も抑える）。引用マーカーが必要（研究や RAG への投入に最適）なら、一言でモードを切り替えられます。

### ログイン済みページのスクレイプ（2FA があっても）

> あなた：「自分の Jira の ToDo を見せて」→ ログイン済みページのスナップショット

**ローカルでログイン済みの Chrome** を再利用——2FA は一度だけ自分で通し、あとは Lasso が引き受けます。プライベート GitHub リポジトリ、社内ネット、有料購読コンテンツなどでも機能します。

> 🔴 **レッドライン**：Lasso は **2FA / SMS 認証 / CAPTCHA / マジックリンクを代わりに解除することは絶対にありません**。これらは必ずローカルの Chrome で一度だけ自分で通してください。

### 生バイトの取得（最速・最安）

> あなた：「この JSON エンドポイントを GET して」→ 生バイトデータ

ページ全体をレンダリングする必要がない場合、直接 HTTP で取得する方がブラウザ経由より **約 4 倍速く、約 4 倍安く** すみます。コンテンツタイプ（JSON / テキスト / バイナリ）は自動検出されます。

### スクリーンショット / アーカイブ

> あなた：「ページ全体のスクリーンショットを撮って」/「PDF で保存して」→ ディスク上のファイルパス

画像も PDF も **ディスクに保存され、パスが返されます**——チャットに巨大なデータが流れ込んでコンテキストを無駄遣いすることはありません。

### ページが読み込むものを可視化

> あなた：「このページが読み込んだサードパーティのトラッカーを教えて」→ トラッカードメインごとの集計付きリソース一覧

ページが読み込むすべてのリソースを自動で識別し、サードパーティドメインごとにグループ化——プライバシーリスクやパフォーマンスのボトルネック発見に重宝します。

### ネイティブデスクトップアプリの操作

> あなた：「ダークモードをオフにして」/「Mail の受信トレイの一番上のメールを読んで」→ 自動操作（検証付き）

macOS では Finder / Mail / Safari / メモ / システム設定など、すべてのネイティブアプリを操作できます。**Windows と Linux でも動作します**（下記の正直な境界線を参照）。すべての操作は検証され——「本当に実行された」ことを確認し、成功を偽ることはありません。

> **正直な境界線**：macOS は実機で検証済み、Windows / Linux はコンパイル時と契約レベルのセルフチェックを通過していますが、実機での網羅的手動テストはまだ進行中です。**「Win/Linux で完全検証済み」とは偽りません。**

### ウェブとデスクトップをまたぐ統合スケジューリング

> あなた：「今操作できるものをすべてリストアップして」→ 統合された一つのリスト

ウェブページとデスクトップウィンドウが一つのリストを共有——「これはブラウザの中」「これはデスクトップ」と区別する必要はありません。Claude が何を操作すべきかを選び、そこからすべてが流れていきます。

### リンク切れの復元

> あなた：「このリンク 404 だ」→ Internet Archive の最新スナップショット

Internet Archive（Wayback Machine）に飛んで、その URL の最後にアーカイブされたコピーを探します。**生きているリンクを切れたとみなすことは絶対にありません**——「これは消えた」とあなたが言った時だけ探しに行きます。

### アンチボット回避（デフォルトはオフ）

> あなた：「このサイト Cloudflare がいてスクレイプできない」→ クラウド Chrome のアンチボット

**デフォルトでは完全にオフ。** あなたが明示的に有効化し、かつクラウドブラウザのキーを設定した場合にだけ動作します。通常のページには不要です。

## インストール

**前提条件**：Node.js ≥ 20、Claude Code（または MCP 対応の任意のクライアント）。

```bash
# Claude Code（推奨）
claude mcp add lasso -- npx -y lasso-mcp
```

Claude Code を再起動 → `/mcp` → `lasso ✓ Connected`。**これで完了——インストールコマンドにキーは不要です。** ブラウジング / スクリーンショット / PDF / デスクトップ操作は即座に使えます（検索だけが例外——[設定](#設定) を参照）。

**デスクトップ操作を使いたい macOS ユーザー**：`lasso doctor` を一度実行し、案内に従って「システム設定 → プライバシーとセキュリティ」の **アクセシビリティ** と **画面収録** の両方で `lasso-rust-helper` にチェックを入れてください（`doctor` が案内するので、自分でパスを探す必要はありません）。

## 設定

**インストールは設定ゼロ**——上記のインストールコマンドだけで、ブラウジング / 取得 / スクリーンショット / PDF / サードパーティリソースの検査 / デスクトップ操作が有効になります。**キーが必要なのは検索だけです。**

### やりたいことから調べる

| やりたいこと | 設定するもの | 解锁される機能 |
|---|---|---|
| 公開ページのスクレイプ / スクリーンショット / PDF / トラッカー確認 / 生バイト取得 / デスクトップ操作 | **何もなし** | インストール直後に動作 |
| 検索 | Zhipu のキー一つ（無料で取得可） | メインの検索エントリー |
| 検索失敗をほぼゼロに（マルチソース化） | Brave / Bing のキーを追加（どちらも無料枠あり） | 一つがダウンしても自動切り替え——ユーザーは何も感じない |
| ログイン済みページのスクレイプ | `lasso launch-chrome` を一度だけ実行 | ローカルの Chrome セッションを再利用 |
| macOS デスクトップの操作 | `lasso doctor` を一度だけ実行 | ネイティブアプリを操作 |
| Cloudflare 保護下のサイトのスクレイプ | 二重確認 + クラウドキー | デフォルトはオフ——明示的な有効化が必要 |

以下では、4 つのモジュールを「とりあえず動く」までの最短経路で紹介します。

### 1. 検索（✅ 無料 · 無料枠あり；キー一つで起動、3 つでほぼゼロ失敗）

**機能**：あらゆるものを検索し、構造化された結果（タイトル、スニペット、リンク）を返します。

**キーは必要か**：はい——Zhipu のキー一つ（無料で取得可）で十分です。

**設定方法**：

```bash
lasso config init        # ~/.lasso/config.json のテンプレートを作成
```

`~/.lasso/config.json` を開いて以下を記入：

```json
{
  "ZHIPU_API_KEY": "your_zhipu_key"
}
```

**より堅牢にしたい場合**（強く推奨）：Brave と Bing も追加——どちらも無料枠があります。いずれかのソースがレート制限や障害で使えなくなっても、自動で次に切り替わります：

```json
{
  "ZHIPU_API_KEY": "your_zhipu_key",
  "BRAVE_API_KEYS": "bravekey1,bravekey2,bravekey3",
  "BING_API_KEYS": "bingkey1,bingkey2"
}
```

> 複数キーはカンマ区切り——N 個のキーで無料枠が N 倍に、自動ローテーションされます。

キー名は上の表の記載と一致しています——そのまま記入してください。保存すれば、次回起動時に Lasso が読み込みます。

**キーの取得方法、無料枠の上限、複数キーのローテーションの詳細** → [キー設定ガイド · 検索](./doc/KEY-GUIDE.md#a-搜索) を参照。

### 2. ログイン済みページのスクレイプ（✅ 無料 · キー不要、コマンド一回だけ）

**機能**：ログイン済みのページ——Jira の ToDo、プライベート GitHub リポジトリ、社内ネット、有料購読コンテンツ——をスクレイプします。

**キーは必要か**：いいえ。

**設定方法**：以下のコマンドを一度だけ実行。ローカルの Chrome を自動検出し、ログイン済みのすべてのセッション（2FA を自分で通したものも含む）を再利用します：

```bash
lasso launch-chrome
```

その後、Claude に「ログイン済みの Jira を開いて」と言えば自動で繋がります。

> 🔴 **レッドライン**：2FA / SMS 認証 / CAPTCHA / マジックリンク——Lasso はこれらを代わりに解除することは絶対にありません。必ずローカルの Chrome で一度だけ自分で通してください。

**詳細** → [キー設定ガイド · ログイン済みブラウジング](./doc/KEY-GUIDE.md#b-登录态浏览命令行配置无-key)。

### 3. デスクトップの操作（✅ 無料 · キー不要、OS で一度だけ認可）

**機能**：macOS / Windows / Linux のネイティブアプリを操作（クリック、入力、ウィンドウ内容の読み取り、ホットキーの実行）します。

**キーは必要か**：いいえ。

**設定方法**（OS を選んでください）：

- **macOS**：`lasso doctor` を一度実行し、案内に従って「システム設定 → プライバシーとセキュリティ」の **アクセシビリティ** と **画面収録** の両方で `lasso-rust-helper` にチェックを入れます。`doctor` が手引きするのでパスを探す必要はありません。
- **Windows**：Claude にデスクトップ操作を依頼した初回に、システムから認可プロンプトが表示されるので「許可」をクリック（macOS のアクセシビリティに相当）。
- **Linux**：アクセシビリティインターフェースがインストールされていることを確認（大半の GNOME / MATE デスクトップには標準で入っています。入っていなければ `sudo apt install at-spi2-core`）。

> **正直な境界線**：macOS は実機で検証済み、Windows / Linux はコンパイル時と契約レベルのセルフチェックを通過していますが、実機での網羅的手動テストはまだ進行中です。**「Win/Linux で完全検証済み」とは偽りません。**

**詳細** → [キー設定ガイド · デスクトップ操作](./doc/KEY-GUIDE.md#c-桌面控制系统授权无-key)。

### 4. クラウドアンチボット（⚠️ 有料・デフォルトはオフ · 二重確認が必要）

**機能**：Cloudflare や厳格なアンチボット保護で守られたサイトをスクレイプします。

**キーは必要か**：はい——そして **あなたが明示的に有効化した時だけ** 動作します。

**設定方法**：以下の 2 つの条件を同時に満たす必要があります：

1. マスタースイッチ：`LASSO_ALLOW_CLOUD_BROWSER` を `true` に設定
2. クラウドキーを少なくとも一つ（browserbase または stagehand——どちらか一つ）

`~/.lasso/config.json` に記入：

```json
{
  "LASSO_ALLOW_CLOUD_BROWSER": true,
  "BROWSERBASE_API_KEY": "your_browserbase_key"
}
```

> デフォルトはオフ——設定しなければこの機能は存在しません。通常のページには不要ですし、**あなたが明示的に有効化した時だけ動作します**。

**クラウドキーの取得方法、トライアル枠** → [キー設定ガイド · クラウドブラウザ](./doc/KEY-GUIDE.md#d-云浏览器反爬默认关双重解锁) を参照。

<details>
<summary><b>高度なチューニング（任意——通常のユーザーは読み飛ばして OK）</b></summary>

日常利用では以下は **完全に無視して構いません**。これらは特殊シナリオ向けで、大半は `lasso config init` で `~/.lasso/config.json` に設定できるか、環境変数で上書きできます（環境変数は設定ファイルより優先されるので、一時的な差し替えに便利です）：

- ログイン済み Chrome のデバッグポートを変更（デフォルトの `9222` が取られている場合）
- キャッシュや状態ファイルの保存場所を変更
- 無料の検索ソースのみに制限
- 社内ネットや特定のプロキシレンジを許可
- ログイン Cookie を暗号化する独自パスフレーズを設定（未設定時は macOS Keychain を使用）
- 検索結果のスナップショットをディスクに保存（回帰テスト用）

変数の完全なリストとデフォルト値：[キー設定ガイド · 高度なチューニング](./doc/KEY-GUIDE.md#e-高级调优可选全不配)。**Surge / Clash の TUN プロキシネットワーク（fake-ip）は初期状態で許可済みです。**

> **後方互換性**：以前 `claude mcp add -e KEY=VAL` でインストールしていた場合、それらの環境変数は **そのまま動作し**、設定ファイルを **上書き** します。設定ファイルは追加の、よりフレンドリーな経路にすぎません——環境変数を置き換えるものではありません。

</details>

## プライバシーとセキュリティ

あなたのデータはあなたのものです。

- **ログイン Cookie は絶対に外部に送出されません**——あなたが明示的に有効化し、ディスクに暗号化する場合を除いて。Lasso があなたのログイン状態をこっそりどこかへ送ることはありません。
- **デスクトップ操作のログはローカルに留まります**——リモートへの報告はゼロ。Lasso はあなたの行動を外に報告しません。
- **クラウドブラウザはデフォルトでオフ**——有効化には **明示的な二重確認**（マスタースイッチ + キー）が必要です。それがない限り、この機能は実質的に存在しません。
- **2FA / CAPTCHA / 認証コードの代行解除はしません**（レッドライン）。これらは必ずローカルのブラウザで、あなた自身が一度だけ通す必要があります。
- **外部者が社内サービスを叩くことはできません**——社内ネットへのアクセスはデフォルトで拒否。Surge / Clash の TUN プロキシネットワークは初期状態で許可済みです。
- **検索結果はデフォルトでディスクに書き込まれません**——記録モード（回帰テスト用）を明示的に有効化した場合のみ。

## トラブルシューティング

**どんな問題でも、最初の一歩は必ず `lasso doctor` です。** セルフチェックを行い、設定の不備を指摘してくれます。

| 症状 | 対処 |
|---|---|
| macOS のデスクトップ操作が動かない | 「システム設定 → プライバシーとセキュリティ → アクセシビリティ / 画面収録」で `lasso-rust-helper` にチェックを入れる（`lasso doctor` が案内します） |
| ログイン済みページのスクレイプが失敗する | ローカルの Chrome で一度手動でログイン（2FA も自分で通す）してから、「ログイン済みの X を開いて」と言う |
| PDF 保存が失敗する | 代わりに「このページの全体スクリーンショットを撮って」と言う |
| 検索がいつも空で返ってくる | キーの有効期限や無料枠の使い切りを確認。複数プロバイダー（Zhipu + Brave + Bing）を追加すると失敗率が劇的に下がります |
| リンクが開かない | 「このリンク切れてるからアーカイブを探して」と言って Internet Archive を確認 |
| 社内ネットへのアクセスがブロックされたと表示される | URL を再確認。TUN プロキシネットワークはデフォルトで許可されていますが、その他の社内ネットには明示的な許可が必要です |

完全な FAQ とデバッグのヒントは [`doc/TROUBLESHOOTING.md`](./doc/TROUBLESHOOTING.md) にあります。

## こんな人に向け・むいていない人

**むいている人**

- **Claude Code をヘビーに使う人**——毎週検索・スクレイプ・デスクトップ操作をし、用途ごとに別の MCP を入れたくない人
- **研究者 / レポート作成者 / データ系の人**——検索から整ったテキストの取得、リンク切れの復元までを一気通貫で
- **RAG / ナレッジベースを構築する人**——ウェブページを整った markdown + 引用マーカー付きに変換し、トークンも費用も節約
- **自動化 / DevOps 系の人**——macOS のネイティブアプリを操作し、ログイン済みの社内ダッシュボードをスクレイプ
- **ログイン済みページをよくスクレイプする人**——ローカルの Chrome セッションを再利用、設定に認証情報を再保存する必要なし

**むいていない人**

- **Claude Code やその他 MCP クライアントを使っていない人**——Lasso は MCP サービスであり、動作には MCP クライアントが必要です
- **単一の機能だけが必要で、すでに専用ソリューションを持っている人**——オールインワンは冗長になる可能性があります
- **2FA / CAPTCHA の回避を探している人**——レッドラインです。やらないし、これからもやりません
