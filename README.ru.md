## 💰 Стоимость кратко

Сам Lasso **полностью бесплатен + MIT open source**. Вот во сколько реально обходится каждая возможность:

| Возможность | Стоимость | Примечания |
|---|---|---|
| Сам Lasso (MCP-сервер + все основные возможности) | ✅ Бесплатно | MIT open source, бесплатно навсегда |
| Поиск (Zhipu + Brave + Bing) | ✅ Есть бесплатный лимит | Zhipu тарифицируется по токенам; Brave **2 000 запросов/мес бесплатно**, Bing **1 000 запросов/мес бесплатно** — можно пользоваться без оплаты |
| Скрейпинг публичных страниц / скриншоты / PDF / аудит сети / сырые байты | ✅ Бесплатно | Работает локально, без ключа, без оплаты |
| Скрейпинг залогиненных страниц (переиспользование локального Chrome) | ✅ Бесплатно | Работает локально, без ключа, без оплаты |
| Управление рабочим столом (macOS / Windows / Linux) | ✅ Бесплатно | Сборка и запуск локально, нужна только авторизация ОС; **опционально** Apple Developer account \$99/год для подписанной постоянной авторизации (работает и без подписи — просто повторно авторизуйтесь каждый раз) |
| Облачный браузер (browserbase / stagehand) | ⚠️ Платно, по умолчанию выключено | Платно после триала; **если не настраивать — не стоит ни копейки** — единственный платный пункт в Lasso |

> Одной фразой: **пока вы не включите облачный браузер, Lasso стоит ноль** — у поиска достаточно бесплатные лимиты для повседневного использования, а всё остальное полностью бесплатно.

## Старт за 60 секунд

### 30 секунд · Установка в одну строку (нулевая конфигурация)

```bash
claude mcp add lasso -- npx -y lasso-mcp
```

Перезапустите Claude Code → введите `/mcp` → увидите `lasso ✓ Connected`. Готово. **В команде установки нет никаких ключей** — настройка идёт отдельным шагом (следующий уровень).

### 30 секунд · Без всякой настройки уже можно делать всё это

Сразу после установки ключи не нужны (это **Уровень 1: нулевая конфигурация**):

- Собрать текст любой **публичной веб-страницы** с конвертацией в чистый markdown
- **Полностраничные скриншоты** и **сохранение в PDF** с возвратом пути к файлу
- Посмотреть, **какие сторонние трекеры загружает страница**
- Забрать сырые байты напрямую из JSON API или файла
- Управлять нативными приложениями macOS (Finder / Mail / «Системные настройки» и т. п. — нужен разовый клик в «Системных настройках»)

Ваш первый результат — просто скажите Claude:

> «Собери текст с example.com и преврати его в markdown»

### Хотите большего? Добавьте в файл конфигурации (Уровень 2)

- **Поиск** → выполните `lasso config init`, чтобы создать `~/.lasso/config.json`, затем впишите ключ Zhipu (см. [Настройка](#настройка))
- **Скрейпинг залогиненных страниц** (Jira / приватный GitHub / корпоративный интранет) → один раз выполните `lasso launch-chrome`
- **Управление рабочим столом macOS** → один раз выполните `lasso doctor`, чтобы пройти авторизацию подсказками

Как получить каждый ключ и какие есть бесплатные лимиты — см. [**Руководство по настройке ключей**](./doc/KEY-GUIDE.md).

## Что он может для вас сделать

Сгруппировано по **тому, что вы хотите сделать**, а не по названию инструмента. Каждое — одна фраза на вход, одна на выход.

### Поиск

> Вы: «Найди X» → структурированные результаты поиска

По умолчанию Zhipu (силен для китайского); можно добавить Brave и Bing для нескольких источников. **Если один источник упёрся в лимит или недоступен — автоматически переключается на следующий, вы ничего не замечаете.** Исчерпание бесплатной квоты у одного провайдера не ломает всю систему.

### Скрейпинг публичных страниц (без логина)

> Вы: «Собери текст с example.com» → чистый текст статьи, три уровня детализации

Автоматически удаляет панели навигации, рекламу, сайдбары и прочий мусор — **экономит 30–70% токенов** (и денег). Нужны маркеры цитирования (отлично подходит для исследований, для подачи в RAG)? Одна фраза переключает режим.

### Скрейпинг залогиненных страниц (даже с 2FA)

> Вы: «Покажи мои задачи в Jira» → снимок залогиненной страницы

Переиспользует **ваш локально залогиненный Chrome** — 2FA вы проходите один раз; дальше Lasso берёт всё на себя. Работает для приватных репозиториев GitHub, корпоративных интранетов, платного контента по подписке и т. д.

> 🔴 **Красная линия**: Lasso **никогда не решает за вас 2FA / SMS-коды / CAPTCHA / magic links**. Вы обязаны один раз вручную пройти их в своём локальном Chrome.

### Получение сырых байтов (быстрее всего и дешевле всего)

> Вы: «Сделай GET этого JSON endpoint» → сырые байты

Когда не нужно рендерить всю страницу, прямой HTTP **примерно в 4 раза быстрее и в 4 раза дешевле**, чем через браузер. Автоматически определяет тип контента (JSON / текст / бинарник).

### Скриншот / Архив

> Вы: «Сделай скриншот всей страницы» / «Сохрани в PDF» → путь к файлу на диске

Все изображения и PDF **сохраняются на диск и возвращается путь** — никакой гигантский блоб не сваливается в чат и не тратит контекст.

### Что загружает страница

> Вы: «Какие сторонние трекеры загрузила эта страница?» → список ресурсов с подсчётом по доменам трекеров

Автоматически определяет все ресурсы, которые грузит страница, и группирует их по сторонним доменам — удобно для оценки рисков приватности и узких мест производительности.

### Управление нативными приложениями рабочего стола

> Вы: «Выключи тёмную тему» / «Прочитай первый элемент во входящих Mail» → автоматическое действие (с проверкой)

На macOS можно управлять Finder / Mail / Safari / Notes / «Системными настройками» и любым нативным приложением. **Windows и Linux тоже работают** (см. честную границу ниже). Каждое действие проверяется — подтверждается «это действительно произошло», успех никогда не подделывается.

> **Честная граница**: macOS проверен на реальном железе; Windows / Linux проходят проверки на этапе компиляции и на уровне контрактов, но полное ручное тестирование на реальных машинах ещё в работе. **Мы не выдаём желаемое за действительное про «полностью проверено на Win/Linux».**

### Единое планирование между вебом и рабочим столом

> Вы: «Покажи всё, чем я сейчас могу управлять» → единый список

Веб-страницы и окна рабочего стола находятся в одном списке — не нужно различать «это в браузере» и «это на рабочем столе». Claude выбирает, с чем работать, и всё идёт оттуда.

### Оживление мёртвых ссылок

> Вы: «Эта ссылка 404» → самый свежий снимок Internet Archive

Идёт в Internet Archive (Wayback Machine), чтобы найти последнюю заархивированную копию этого URL. **Никогда не считает живую ссылку мёртвой** — ищет только когда вы говорите «этого больше нет».

### Обход анти-бота (по умолчанию выключено)

> Вы: «Тут Cloudflare, не могу спарсить» → анти-бот через облачный Chrome

**Полностью выключено по умолчанию.** Активируется только когда вы явно его включаете И настроили ключ облачного браузера. Для обычных страниц он не нужен.

## Установка

**Предварительные требования**: Node.js ≥ 20; Claude Code (или любой клиент с поддержкой MCP).

```bash
# Claude Code (рекомендуется)
claude mcp add lasso -- npx -y lasso-mcp
```

Перезапустите Claude Code → `/mcp` → `lasso ✓ Connected`. **Это всё — в команде установки нет ключей.** Скрейпинг / скриншоты / PDF / управление рабочим столом работают сразу (единственное исключение — поиск, см. [Настройка](#настройка)).

**Пользователям macOS, которым нужно управление рабочим столом**: один раз выполните `lasso doctor` и по подсказкам отметьте `lasso-rust-helper` в «Системные настройки → Конфиденциальность и безопасность» в разделах **Универсальный доступ** и **Запись экрана** (`doctor` проведёт вас за руку — путь искать не нужно).

## Настройка

**Установка — без конфигурации** — приведённая выше команда установки уже активирует скрейпинг / забор байтов / скриншоты / PDF / аудит сторонних ресурсов / управление рабочим столом. **Только для поиска нужен ключ.**

### Сверьтесь с тем, что хотите сделать

| Что вы хотите | Что настроить | Что это открывает |
|---|---|---|
| Скрейпинг публичных страниц / скриншоты / PDF / просмотр трекеров / сырые байты / управление рабочим столом | **Ничего** | Работает сразу после установки |
| Поиск | Один ключ Zhipu (бесплатно оформить) | Основная точка входа для поиска |
| Почти нулевой процент failures поиска (несколько источников) | Добавить ключи Brave / Bing (у обоих есть бесплатные лимиты) | Автоматическое переключение при сбое одного — вы ничего не замечаете |
| Скрейпинг залогиненных страниц | Один раз выполнить `lasso launch-chrome` | Переиспользует сессию вашего локального Chrome |
| Управление рабочим столом macOS | Один раз выполнить `lasso doctor` | Управление нативными приложениями |
| Скрейпинг сайтов под Cloudflare | Двойное подтверждение + облачный ключ | По умолчанию выключено; нужно явное включение |

Ниже каждый из четырёх модулей разобран самым коротким путём к «просто работает».

### 1. Поиск (✅ Бесплатно · бесплатный лимит; один ключ для старта, три — почти нулевой процент сбоев)

**Что делает**: ищет что угодно, возвращает структурированные результаты (заголовок, сниппет, ссылка).

**Нужен ли ключ**: Да — достаточно одного ключа Zhipu (оформляется бесплатно).

**Как настроить**:

```bash
lasso config init        # создаёт шаблон ~/.lasso/config.json
```

Откройте `~/.lasso/config.json` и впишите:

```json
{
  "ZHIPU_API_KEY": "your_zhipu_key"
}
```

**Хотите надёжнее** (настоятельно рекомендуется): добавьте также Brave и Bing — у обоих есть бесплатные лимиты. Если один источник упирается в лимит или недоступен, происходит автоматическое переключение на следующий — вы ничего не замечаете:

```json
{
  "ZHIPU_API_KEY": "your_zhipu_key",
  "BRAVE_API_KEYS": "bravekey1,bravekey2,bravekey3",
  "BING_API_KEYS": "bingkey1,bingkey2"
}
```

> Несколько ключей разделяются запятыми — N ключей дают в N раз больше бесплатной квоты с автоматической ротацией.

Названия ключей совпадают с теми, что в таблице выше, — просто впишите их. Сохраните файл; Lasso подхватит его при следующем запуске.

**Как оформлять ключи, размеры бесплатных квот, детали ротации нескольких ключей** → см. [Руководство по настройке ключей · Поиск](./doc/KEY-GUIDE.md#a-搜索).

### 2. Скрейпинг залогиненных страниц (✅ Бесплатно · без ключа, одна команда)

**Что делает**: скрейпит страницы, на которые вы залогинены, — задачи в Jira, приватные репозитории GitHub, корпоративные интранеты, платный контент по подписке.

**Нужен ли ключ**: Нет.

**Как настроить**: Один раз выполните команду ниже. Она автоматически находит ваш локальный Chrome и переиспользует все сессии, в которые вы уже залогинились (включая те, где вы сами прошли 2FA):

```bash
lasso launch-chrome
```

После этого скажите Claude «открой мой залогиненный Jira» — и всё подключится автоматически.

> 🔴 **Красная линия**: 2FA / SMS-коды / CAPTCHA / magic links — Lasso никогда не решает их за вас. Вы обязаны один раз вручную пройти их в своём локальном Chrome.

**Подробнее** → [Руководство по настройке ключей · Залогиненный серфинг](./doc/KEY-GUIDE.md#b-登录态浏览命令行配置无-key).

### 3. Управление рабочим столом (✅ Бесплатно · без ключа, разовая авторизация в ОС)

**Что делает**: управляет нативными приложениями на macOS / Windows / Linux (клик, ввод, чтение содержимого окон, хоткеи).

**Нужен ли ключ**: Нет.

**Как настроить** (выберите свою ОС):

- **macOS**: Один раз выполните `lasso doctor` и по подсказкам отметьте `lasso-rust-helper` в «Системные настройки → Конфиденциальность и безопасность» в разделах **Универсальный доступ** и **Запись экрана**. `doctor` проведёт вас за руку — путь искать не нужно.
- **Windows**: При первом запросе к Claude сделать действие на рабочем столе система покажет запрос на авторизацию — нажмите «Разрешить» (аналог macOS Универсального доступа).
- **Linux**: Убедитесь, что установлен интерфейс доступности (на большинстве рабочих столов GNOME / MATE он есть по умолчанию; если нет — `sudo apt install at-spi2-core`).

> **Честная граница**: macOS проверен на реальном железе; Windows / Linux проходят проверки на этапе компиляции и на уровне контрактов, но полное ручное тестирование на реальных машинах ещё в работе. **Мы не выдаём желаемое за действительное про «полностью проверено на Win/Linux».**

**Подробнее** → [Руководство по настройке ключей · Управление рабочим столом](./doc/KEY-GUIDE.md#c-桌面控制系统授权无-key).

### 4. Облачный анти-бот (⚠️ Платно, по умолчанию выключено · нужно двойное подтверждение)

**Что делает**: скрейпит сайты под защитой Cloudflare или серьёзным анти-ботом.

**Нужен ли ключ**: Да — и активируется только когда **вы явно его включаете**.

**Как настроить**: Должны одновременно выполняться оба условия:

1. Главный переключатель: установите `LASSO_ALLOW_CLOUD_BROWSER` в `true`
2. Хотя бы один облачный ключ (browserbase или stagehand — выберите один)

Впишите в `~/.lasso/config.json`:

```json
{
  "LASSO_ALLOW_CLOUD_BROWSER": true,
  "BROWSERBASE_API_KEY": "your_browserbase_key"
}
```

> По умолчанию выключено — нет конфигурации, нет и возможности. Для обычных страниц это не нужно, **и активируется только при явном включении**.

**Как оформлять облачные ключи, триальные квоты** → см. [Руководство по настройке ключей · Облачный браузер](./doc/KEY-GUIDE.md#d-云浏览器反爬默认关双重解锁).

<details>
<summary><b>Тонкая настройка для продвинутых (опционально — обычным пользователям можно пропустить)</b></summary>

Для повседневного использования всё ниже можно **полностью игнорировать**. Это только для специфических сценариев, и большую часть можно задать через `lasso config init` в `~/.lasso/config.json` или переопределить через переменные окружения (переменные окружения имеют приоритет над файлом конфигурации — удобно для временных замен):

- Сменить отладочный порт залогиненного Chrome (когда дефолтный `9222` занят)
- Перенести файлы кеша / состояния в другое место
- Ограничиться только бесплатными источниками поиска
- Разрешить корпоративный интранет / особые диапазоны прокси
- Задать собственную парольную фразу для шифрования login-куки (если не задана — используется macOS Keychain)
- Сохранять снимки результатов поиска на диск (для регрессионного тестирования)

Полный список переменных и значения по умолчанию: [Руководство по настройке ключей · Тонкая настройка](./doc/KEY-GUIDE.md#e-高级调优可选全不配). **Прокси-сети Surge / Clash TUN (fake-ip) уже разрешены из коробки.**

> **Обратная совместимость**: если вы ранее устанавливали через `claude mcp add -e KEY=VAL`, эти переменные окружения **продолжают работать** и **переопределяют** файл конфигурации. Файл конфигурации — это просто дополнительный, более дружелюбный путь, он не заменяет переменные окружения.

## Конфиденциальность и безопасность

Ваши данные принадлежат вам.

- **Login-куки никогда не экспортируются**, если только вы явно не согласились и не зашифровали их на диск. Lasso никогда втайне никуда не отправляет ваше залогиненное состояние.
- **Логи действий на рабочем столе хранятся локально** — ноль удалённой отчётности. Lasso не «звонит домой» о том, что вы делаете.
- **Облачный браузер по умолчанию выключен** — для активации нужно ваше **явное двойное подтверждение** (главный переключатель + ключ). Без него возможность фактически не существует.
- **Никакого решения 2FA / CAPTCHA / кодов подтверждения** (красная линия). Они всегда требуют, чтобы вы лично один раз прошли их в своём локальном браузере.
- **Посторонние не могут дотянуться до ваших внутренних сервисов** — доступ к внутренней сети по умолчанию запрещён; прокси-сети Surge / Clash TUN уже разрешены из коробки.
- **Результаты поиска по умолчанию не пишутся на диск** — только если вы явно включите режим записи (для регрессионного тестирования).

## Устранение неполадок

**При любой проблеме первый шаг — всегда `lasso doctor`.** Он сам себя проверяет и говорит, что настроено не так.

| Симптом | Что делать |
|---|---|
| Не работает управление рабочим столом macOS | Отметьте `lasso-rust-helper` в «Системные настройки → Конфиденциальность и безопасность → Универсальный доступ / Запись экрана» (`lasso doctor` проведёт вас) |
| Не удаётся скрейпить залогиненную страницу | Один раз вручную залогиньтесь в локальном Chrome (пройдите и 2FA), затем скажите «открой мой залогиненный X» |
| Не работает сохранение в PDF | Скажите вместо этого «сделай скриншот всей страницы» |
| Поиск постоянно возвращает пустоту | Проверьте, не истёк ли ключ / не исчерпана ли квота; добавление нескольких провайдеров (Zhipu + Brave + Bing) резко снижает процент сбоев |
| Ссылка не открывается | Скажите «эта ссылка мёртвая, найди архив» — проверим в Internet Archive |
| Пишет, что доступ к внутренней сети заблокирован | Перепроверьте URL; TUN-прокси-сети разрешены по умолчанию, для остальных внутренних сетей нужно явное разрешение |

Полный FAQ и советы по отладке — в [`doc/TROUBLESHOOTING.md`](./doc/TROUBLESHOOTING.md).

## Кому подходит / Кому не подходит

**Подходит**

- **Активным пользователям Claude Code** — каждую неделю ищут, скрейпят и управляют рабочим столом и не хотят ставить отдельный MCP под каждый случай
- **Исследователям / авторам отчётов / дата-инженерам** — поиск, сбор чистого текста, оживление мёртвых ссылок от начала до конца
- **Тем, кто строит RAG / базы знаний** — веб-страницы в чистый markdown с маркерами цитирования, экономия токенов и денег
- **Специалистам по автоматизации / DevOps** — управление нативными приложениями macOS, скрейпинг залогиненных внутренних дашбордов
- **Всем, кто часто скрейпит залогиненные страницы** — переиспользование сессии локального Chrome, без повторного хранения учётных данных в конфиге

**Не подходит**

- **Тем, кто не использует Claude Code или другой MCP-клиент** — Lasso — это MCP-сервис и требует MCP-клиент, чтобы им управляли
- **Тем, кому нужна лишь одна возможность и уже есть под неё специализированное решение** — «всё в одном» может оказаться избыточным
- **Тем, кто хочет обходить 2FA / CAPTCHA** — красная линия; мы этого не делаем и не будем.
